## 1. Text Mutation Laboratory

### Core Idea

A terminal-based system that mutates text through chained transformations:

- corruption
- symbol substitution
- spacing distortion
- recursive mutation
- entropy experiments

Potential outputs:

- glitch poetry
- pseudo-languages
- corrupted terminal text
- evolving typography

### Reality Check

Highly buildable.

Clear execution path:

    input text → apply mutations → print transformed output

Fast visible feedback.

Very low setup friction.

### Main Risks

The project becomes dangerous if it turns into:

- plugin systems
- generalized transformation frameworks
- DSLs
- GUI editors
- configuration-heavy architecture

### Simplified MVP

Single-file terminal script:

    python mutate.py "hello world"

Apply:

- random symbol swaps
- duplication
- spacing mutation

Print output immediately.

### Feedback Loop

Excellent.

This project produces:

- immediate output
- fast experimentation
- visible iteration
- rewarding progress

### Verdict

BUILD NOW

Strongest balance of:

- creativity
- finishability
- experimentation
- low friction

## 2. Terminal Biome Generator

### Core Idea

Generate procedural ASCII terrain maps inside the terminal.

Possible terrain:

- water
- forest
- desert
- mountains
- swamps

### Reality Check

Buildable — but dangerous if scope expands.

The project becomes risky when it evolves into:

- ecosystem simulation
- pathfinding
- infinite worlds
- AI systems
- survival mechanics

### Hidden Complexity

The real difficulty is not generation.

It is:

- rendering stability
- tuning randomness
- visual readability
- simulation debugging

### Safe MVP

Generate static ASCII maps only.

No simulation.

Example:

    ~ water
    . plains
    T forest
    ^ mountains

### Biggest Risk

Simulation creep.

You will think:

> “it just needs one more system.”

That is where momentum collapses.

### Verdict

BUILD NOW

—but only as a constrained static generator.

## 4. ASCII Fluid Simulator

### Core Idea

Render fluid-like behavior using ASCII characters in the terminal.

### Reality Check

This appears small.

It is not.

This is a debugging-heavy simulation project.

### Hidden Complexity

Main problems:

- simulation stability
- update ordering
- rendering flicker
- timestep bugs
- performance tuning

### Safe MVP

Do NOT build real fluids first.

Start with:

- falling sand
- simple gravity
- basic particle spread

### Biggest Risk

Spending hours debugging unstable behavior with little visible progress.

### Verdict

DELAY

Strong learning value, but dangerous completion risk.
